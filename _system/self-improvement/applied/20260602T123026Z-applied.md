# Self-Improvement Proposal: Make app-identity validators hybrid-layout and camelCase aware

## Reason

Two validators mis-signal on this hybrid-layout repo. (1) check-app-definition-state.sh judges 'defined' only by app/src/, so Budget Beacon (real source under top-level src/ per PROJECT_PROFILE Runtime code roots, hybrid layout) is wrongly reported blank_app_undefined, telling agents to re-interview/scaffold a far-along app. Fix: judge 'defined' by real source in the PROJECT_PROFILE-declared runtime roots, defaulting to app/src. (2) check-project-target-consistency.sh normalize_name turns spaces to hyphens but does not split camelCase, so app name 'Budget Beacon'->budget-beacon never matches repo 'BudgetBeacon'->budgetbeacon, emitting a permanent false-positive warn. Fix: split camelCase boundaries before normalizing. Both fixes are generic (no app facts) and taggable as maintainer-promotion candidates.
