# Roadmap

> **Truth Reset — 2026-04-25.** Previous roadmap claimed "Milestone 1: Initial Release (Completed) — Risks: None active." Audit on 2026-04-25 found multiple mocked surfaces and inflated completion claims across planning docs. The horizons and milestones below replace the prior text. Detail in `docs/COMPLETION_MASTER_PLAN.md`.

## Current horizons

- **Horizon 1 (now → next ~6 weeks): Truth, GUI, and trust.** Align docs with reality, finish the GUI surface end-to-end, make backup/restore/reports trustworthy. Covers M0–M4.
- **Horizon 2 (next quarter): Real data ingestion.** Replace simulated Bank Sync, Scavenge, Credit Fetch, AI chatbot, and Beacon Bridge sync with real adapters or honest "manual" labels behind feature flags. Covers M5–M7.
- **Horizon 3 (release-ready): Deepening & polish.** Tax/credit/debt/household planning deepening, Android/web release polish, install/recovery confidence. Covers M8–M9.

## Milestones

### M0 — Repo Truth Reset (in progress, 2026-04-25)
- **Outcome:** Planning docs match runtime reality. Two new strategy docs in place.
- **Dependencies:** Completed audit recorded in `docs/GUI_COMPLETION_MAP.md`.
- **Risks:** Audit goes stale if routes are edited before M1 starts. Mitigation: re-run before M1 kickoff.

### M1 — GUI Surface Audit & Completion Matrix
- **Outcome:** Per-control checklists for all 15 routes, scored real / partial / mock / missing / broken / polish-needed.
- **Dependencies:** M0 docs landed.
- **Risks:** Scope creep into implementation; M1 must remain a planning artifact.

### M2 — Core Data + Validation Hardening
- **Outcome:** `npm test` exists and runs Vitest. Budget engine, income frequency normalization, JSON backup schema, and CSV parsing have unit tests. Zod schemas guard CRUD boundaries.
- **Dependencies:** M1.
- **Risks:** No existing test files in repo despite Vitest in devDependencies — green-field test setup required.

### M3 — Full GUI Completion Pass
- **Outcome:** Every alert-only button replaced with real action or feature flag. Loading/empty/error states present on every route. Mobile/Android safe-area pass. Error boundaries.
- **Dependencies:** M2 (so tests guard regressions during sweep).
- **Risks:** Largest milestone; risk of partial completion. Slice route-by-route, commit per route.

### M4 — Reports, Backup, Restore, Recovery
- **Outcome:** Real CSV/JSON export with versioned schema. Restore-confirmation flow. Round-trip integrity test passes. Reports export → real file (not `alert()`).
- **Dependencies:** M3 (so reports surfaces are stable).
- **Risks:** JSON schema versioning missed → backups created now break later.

### M5 — Ledger + Bank/Data Import Foundation
- **Outcome:** CSV import works end-to-end with dedupe and review queue. QFX/OFX parser scaffolded. Mock Bank Sync removed or hidden behind disabled feature flag.
- **Dependencies:** M4 (export/import schemas stable).
- **Risks:** Aggregator (Plaid/MX/Finicity) decision deferred; do not commit to provider until needed.

### M6 — Vault + OCR + Extraction Review
- **Outcome:** Tesseract.js (or equivalent) browser-side OCR. Extraction-review UI. Apply-on-approve flow writes to entities with provenance link to source document. Mock Scavenge removed.
- **Dependencies:** M5 (review-queue pattern established for transactions can be reused for documents).
- **Risks:** OCR accuracy on real documents — must surface confidence scores and never auto-commit.

### M7 — AI Assistant Real Integration
- **Outcome:** Provider abstraction with local (Ollama / OpenAI-compatible endpoint) first, cloud second. `aiConfig` persists. Chatbot responses generated by configured provider. Action proposals require confirmation; no silent writes.
- **Dependencies:** M6 (Vault provides document context for assistant).
- **Risks:** Privacy — cloud provider must be opt-in and clearly labeled; default is local-only.

### M8 — Tax / Credit / Debt / Household Planning Deepening
- **Outcome:** Tax forms either real (proper field set + calculations) or labeled "manual draft assist." Avalanche vs snowball debt comparison. Credit snapshot UX honest about manual entry. Cross-module household planning summary.
- **Dependencies:** M7.
- **Risks:** Tax compliance accuracy — if not real, must be unambiguously labeled draft.

### M9 — Android / Web Final Polish + Release Candidate
- **Outcome:** Safe-area pass on Android. PWA install flow validated. Desktop/web parity decisions documented. Install/update/recovery docs in repo. Release checklist passes.
- **Dependencies:** M8.
- **Risks:** Capacitor / Android version drift between development and release; pin versions.

## Cross-cutting themes

- **Honesty over breadth.** Every milestone is allowed to remove or feature-flag a fake surface; that counts as progress, not regression.
- **Backup safety first.** No destructive testing without verified backup round-trip (M4 onward).
- **Local-first default.** Cloud features (AI, aggregator) are always opt-in and clearly labeled.
